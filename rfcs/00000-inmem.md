## Proposal

**Main Invariant**: During search or operations that need to read chunks of data, we first create an "accessor" that registers itself with the current, monotonically increasing generation.
As long as this accessor is alive, if it accesses a vector with an associated generation **less than or equal to** the current generation (with acquire semantics), then it **must** be safe to access the associated data payload.

The rest of this proposal discusses how we maintain all of these invariants.

### Default State

In the default state, the tags for data are set to `u62::MAX | Available` in the source-of-truth registry.

### Writing Data

Data may be written upon a successful `CAS` from `u62::MAX | Available` to `u62::MAX | Owned`.
If the data is in a state where a transition to `u62::MAX | Available` is valid, then we can short-ciruit the transition to `u62::MAX | Owned` if that ends up being more efficient.

Once data has been written, it is released into the wild with a transition to `Current | Used` where `Current` is the current generation.
This makes it immediately available to current readers and thus must be done with Release sementics to uphold the main invariant.

### Deletes

Deletes need to happen in three pieces:

1. The data-to-be-deleted should be atomically CAS from `X | Used` to `u62::MAX | Owned`.
   It then enters a waiting state.
2. Once we can prove that no users of generation `X`, this slot can be made available.
